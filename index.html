算法学习记录-栈与队列

发表于 2023-07-31

本文是对代码随想录第六章（栈与队列）的学习记录。

用栈实现队列[go]

请你仅使用两个栈实现先入先出队列。队列应当支持一般队列支持的所有操作（push、pop、peek、empty）

  • 需要使用两个栈，push操作时将数据放进输入栈，pop操作时，将输入栈数据导出到输出栈，再从输出栈弹出数据
  • 时间复杂度: push/empty O(1) pop/peek O(n)
  • 空间复杂度: O(n)

用队列实现栈[go]

请你仅使用两个队列实现一个后入先出（LIFO）的栈，并支持普通栈的全部四种操作（push、top、pop 和 empty）。

  • 其中一个队列用于备份，弹出操作时，先备份其余元素，弹出队首，再还原
  • 时间复杂度: push/empty O(1) pop/top O(n)
  • 空间复杂度: O(n)

有效的括号[go]

给定一个只包括 '('，')'，'{'，'}'，'['，']' 的字符串 s ，判断字符串是否有效。

  • 由于栈结构的特殊性，非常适合做对称匹配类的题目
  • 不要上来就敲代码，先分析题目中匹配不上的情况：1.括号数量不对 2.括号类型不匹配
  • 时间复杂度: O(n)
  • 空间复杂度: O(n)

删除字符串中的所有相邻重复项[go]

给出由小写字母组成的字符串 S，重复项删除操作会选择两个相邻且相同的字母，并删除它们。在 S 上反复执行重复项删除操作，直到无法继续删除。

  • 匹配问题是栈的强项，栈中匹配完成剩下的数据就是新字符串
  • 时间复杂度: O(n)
  • 空间复杂度: O(n)

逆波兰表达式求值[go]

给你一个字符串数组 tokens ，表示一个根据 逆波兰表示法 表示的算术表达式。
请你计算该表达式。返回一个表示表达式值的整数。

  • 匹配问题是栈的强项，建立一个中间栈存放运算数据，遇到运算符后取两个数进行计算，结果重新入栈
  • 注意算数表达式的运算顺序，栈中先弹出的是右边的值，后弹出的是左边的值
  • 时间复杂度: O(n)
  • 空间复杂度: O(n)

滑动窗口最大值[go]

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

  • 如果使用暴力算法，每次滑动窗口时都要找到最大值，时间复杂度为O(n*k)
  • 可以维护一个单调队列，push时如果value值大于入口元素的值，就把队列中小于value的元素全部弹出（因为滑动窗口是从左向右移动的，所以当出现最大值时，左边的信息可以释放）
  • pop时如果窗口移除的元素等于队列出口，则将出口元素弹出，否则不进行任何操作（因为最大值没有弹出时，滑动窗口中最大值不变）
  • 时间复杂度: O(n)
  • 空间复杂度: O(k)

总结

本章主要介绍了栈和队列的使用，其中：

  • 用栈实现队列、用队列实现栈是对栈和队列结构的表述
  • 有效的括号、删除字符串中的所有相邻重复项、逆波兰表达式求值利用了栈先入后出的特性做匹配

算法学习记录-双指针法

发表于 2023-07-28 更新于 2023-07-31

本文是对代码随想录第五章（双指针法）的学习记录。

移除元素[go]

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

  • 两个指针，从左到右给数组赋值
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

反转字符串[go]

编写一个函数，反转输入的字符串。

  • 没啥说的，两个指针，不断交换两边的字符即可
  • 不要使用库函数
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

替换空格[go]

请实现一个函数，把字符串 s 中的每个空格替换成”%20”。

  • 数组的插入删除操作都非常耗时，所以可以先扩容数组，再从后向前改写每个元素的值
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

反转链表[go]

给你单链表的头节点 head ，请你反转链表，并返回反转后的链表。

  • 从左到右依次反转链表，我愿称之为3指针法
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

删除链表的倒数第N个节点[go]

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

  • 两个指针，right先走n步，然后同时移动left和right，right到末尾后，left就是相应位置
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

链表相交[go]

给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 null 。

  • 求两个链表的长度，计算差值，然后分别移动，节点相等则为交点
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

环形链表II[go]

给定一个链表的头节点 head ，返回链表开始入环的第一个节点。如果链表无环，则返回 null。

  • 快慢指针法，fast移动2个字符，slow移动1个字符，找到相遇节点
  • 从头节点和相遇节点各自出发一个指针，相遇的地方就是入环节点
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

三数之和[go]

给你一个整数数组 nums ，判断是否存在三元组[nums[i], nums[j], nums[k]]满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

  • 先对数组排序，从头结点出发一个指针，尾结点出发一个指针，计算三者之和，>0则right左移，<0则left右移
  • 相比暴力算法降低了一个指数，可以剪枝处理，避免不必要的运算
  • 时间复杂度: O(n^2)
  • 空间复杂度: O(1)

四数之和[go]

给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述全部条件且不重复的四元组 [nums[a], nums[b], nums[c], nums[d]]

  • 思路和三数之和一样，先对数组排序，从头结点出发一个指针，尾结点出发一个指针，计算四者之和，>0则right左移，<0则left右移
  • 相比暴力算法降低了一个指数，可以剪枝处理，避免不必要的运算
  • 时间复杂度: O(n^2)
  • 空间复杂度: O(1)

总结

本章主要介绍了双指针法的题目，基本上换汤不换药，只要分析到位都能做出来，其中：

  • 移除元素、反转字符串、替换空格、反转链表、删除链表的倒数第N个节点、链表相交、环形链表II、三数之和、四数之和都是通过多个指针减少代码中遍历次数

心得：

  • 使用双指针可以提高效率，一般是将O(n^2)的时间复杂度，降为O(n)

  • 在求倒数第X个元素时，可以让其中一个指针先走X步，这样可以减少一次遍历

  • 遍历时尽量使用for循环，用while的话容易把自己绕晕

算法学习记录-字符串

发表于 2023-07-24 更新于 2023-07-31

本文是对代码随想录第四章（字符串）的学习记录。

反转字符串[go]

编写一个函数，反转输入的字符串。

  • 没啥说的，两个指针，不断交换两边的字符即可
  • 不要使用库函数
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

反转字符串2[go]

给定一个字符串 s 和一个整数 k，从字符串开头算起, 每计数至 2k 个字符，就反转这 2k 个字符中的前 k 个字符。
如果剩余字符少于 k 个，则将剩余字符全部反转。
如果剩余字符小于 2k 但大于或等于 k 个，则反转前 k 个字符，其余字符保持原样。

  • 遍历的时候每次移动2k，然后判断需要反转的区间即可
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

替换空格[go]

请实现一个函数，把字符串 s 中的每个空格替换成”%20”。

  • 数组的插入删除操作都非常耗时，所以可以先扩容数组，再从后向前改写每个元素的值
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

反转字符串里的单词[go]

给定一个字符串，逐个翻转字符串中的每个单词。

  • 移除多余空格（使用指针法移除，O(n)）
  • 字符串反转
  • 单词反转
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

左旋转字符串[go]

把字符串前面的若干个字符转移到字符串的尾部。

  • 做三次反转即可（两次局部 + 一次全局）
  • 时间复杂度: O(n)
  • 空间复杂度: O(1)

找出字符串中第一个匹配项的下标[go]

字符串搜索，成功返回下标，失败返回-1，空字符串返回0。

  • 经典的KMP算法，算法思想是失配后模式串右移，避免无用比较
  • 需要一个next数组，记录前后缀相等的最大数
  • 如果匹配，则两个串的指针都+1，如果不匹配，模式串的指针后移到next[i]
  • 令k = 2i - j，则不难发现：while循环每迭代一轮，k都会严格递增
  • 启动时有i = j = 0，即k = 0；算法结束时i <= n且j >=0，故有k<2n。所以while最多执行2n轮。
  • 时间复杂度: O(n+m)
  • 空间复杂度: O(m)

重复的子字符串[go]

给定一个非空的字符串 s ，检查是否可以通过由它的一个子串重复多次构成。

  • 移动匹配法，看字符串s是否由重复子串组成，搜索需要使用KMP算法
  • next数组法，根据最长相等前后缀的规则，当一个字符串由重复子串组成的，最长相等前后缀不包含的子串就是最小重复子串。
  • 数组长度减去最长相同前后缀的长度相当于是第一个周期的长度，如果这个周期可以被整除，就说明整个数组就是这个周期的循环。
  • 时间复杂度: O(n)
  • 空间复杂度: O(n)

总结

本章主要介绍了字符串的题目，其中：

  • 反转字符串、反转字符串2、替换空格、反转字符串里的单词、左旋转字符串本质上都是指针法的应用

  • 找出字符串中第一个匹配项的下标是典型的KMP算法，重复的子字符串利用KMP算法中的next数组获得最大前后缀，最长相等前后缀不包含的子串就是最小重复子串

心得：

  • 串可以看成数组，插入删除操作都非常耗时，所以可以先扩容数组，再依次改写每个元素的值

  • 这几道题都用到了指针，遍历时尽量用for循环，用while写的话很容易把自己搞晕。

  • KMP算法就是构建next数组记录比较失败后，模式串指针下一时刻的位置，使用双指针法比较，避免重复操作
